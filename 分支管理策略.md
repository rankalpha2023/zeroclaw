# ZeroClaw 分支管理策略

## 需求分析

用户希望：
1. 把从 0.3.2 版本开始的所有修改放入一个新的分支
2. 让 master 分支一直跟随 fork 的源项目（zeroclaw-labs/zeroclaw）更新
3. 以后能够对比修改的代码与 master 分支最新版本的区别，进行修正
4. 始终跟着 master 主版本更新

## 解决方案

### 1. 创建新分支保存当前修改

```bash
# 提交当前修改
git add .
git commit -m "feat: 保存当前修改到新分支"

# 创建并切换到新分支（例如：feature/my-changes）
git checkout -b feature/my-changes

# 推送新分支到远程仓库
git push origin feature/my-changes
```

### 2. 重置 master 分支以跟随源项目

```bash
# 切换回 master 分支
git checkout master

# 重置 master 分支到 upstream/master 的最新版本
git reset --hard upstream/master

# 推送到远程仓库（强制推送）
git push origin master --force
```

### 3. 配置远程仓库

已完成：
```bash
# 添加源项目作为 upstream 远程仓库
git remote add upstream https://github.com/zeroclaw-labs/zeroclaw.git

# 拉取 upstream 的最新代码
git fetch upstream
```

### 4. 后续操作流程

#### 定期更新 master 分支

```bash
# 切换到 master 分支
git checkout master

# 拉取 upstream 的最新代码
git fetch upstream

# 合并 upstream/master 到本地 master
git merge upstream/master

# 推送到远程仓库
git push origin master
```

#### 对比修改与最新版本

```bash
# 切换到你的修改分支
git checkout feature/my-changes

# 查看与 master 分支的差异
git diff master

# 或者查看具体文件的差异
git diff master -- src/agent/dispatcher.rs
```

#### 合并最新版本到修改分支

```bash
# 切换到你的修改分支
git checkout feature/my-changes

# 合并 master 分支的最新代码
git merge master

# 解决可能的冲突
# ...

# 提交合并结果
git commit -m "Merge latest changes from master"

# 推送到远程仓库
git push origin feature/my-changes
```

## 分支管理最佳实践

1. **master 分支**：始终保持与 upstream/master 同步，作为最新稳定版本的参考
2. **feature 分支**：用于存放你的修改和新功能开发
3. **定期更新**：至少每周更新一次 master 分支，确保与源项目保持同步
4. **代码审查**：在合并前仔细审查代码差异，确保修改与最新版本兼容
5. **测试**：每次合并后运行测试，确保修改不会破坏现有功能

## 常见问题处理

### 冲突解决

当合并 master 到你的 feature 分支时，可能会遇到冲突。解决方法：

1. 查看冲突文件：`git status`
2. 编辑冲突文件，手动解决冲突
3. 标记冲突已解决：`git add <冲突文件>`
4. 完成合并：`git commit`

### 版本回退

如果需要回退到某个版本：

```bash
# 查看提交历史
git log --oneline

# 回退到指定版本
git reset --hard <commit哈希>
```

### 远程分支管理

查看远程分支：
```bash
git branch -r
```

删除远程分支：
```bash
git push origin --delete <分支名>
```

## 总结

通过以上策略，你可以：
1. 保持 master 分支与源项目同步
2. 在独立的 feature 分支中开发和维护你的修改
3. 定期对比和合并最新版本的代码
4. 确保你的修改始终与源项目的最新版本兼容

这种分支管理方式可以帮助你更有效地跟踪和管理代码变更，同时确保你的修改能够与上游项目的发展保持同步。
